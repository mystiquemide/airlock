The problem

Raw A2A has no membrane.

Agents discover each other and fire tasks over HTTP with zero oversight. Nothing stands between intent and execution.

An agent can wire money

External agents holding payment tools can move funds with no permission scoping and no one watching it happen.

An injected agent can leak data

A prompt-injected agent can exfiltrate customer PII before a human ever sees the request that caused it.

No human in the loop

Agent-to-agent protocols execute tasks the moment they're requested. There's no gate, and no record once it's done.

How it works

One enforcement point. Every request passes through it.

External agents hold no privileged tools. They can only request — the Warden decides, and Band's history writes the record.

Capability separation — privileged tools live only on the Warden
Policy engine — static YAML, fail-closed
Audit — Band history is the system of record
The policy engine

Every request resolves to exactly one verdict.

Three outcomes. No ambiguity. A verdict color only ever means its verdict.

Auto-allow

Low-risk, resolves without a human.

Matches a safe rule — read-only lookups, requests under threshold. Executes immediately.

Cleared. Logged.
Require-human

Risky enough to need a person.

Payouts over threshold, prod writes — @mentions the Compliance Officer in Band and blocks until they reply.

Action requires human clearance: payout($4,200). Approve or deny.
Hard-deny

Forbidden. Never reaches a human.

Matches a hard-deny rule — payout to an unlisted account, anomalous request pattern. Rejected outright.

Blocked by policy: payout.account_not_allowlisted. Not executed.
The compliance artifact

The ledger writes itself.

Every request, verdict, and human decision — pulled straight from Band's immutable history, not a separate store.
